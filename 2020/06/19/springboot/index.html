springboot自动装配

Posted on 2020-06-19 Edited on 2020-06-28

spring boot 三大特性
组件自动装配：web mvc、web flue、jdbc等
嵌入式web容器：tomcat、jetty、undertow
生产准备的特性：指标、健康检查、外部化配置
（cpu、内存）（应用、数据库、磁盘）

组件自动装配
激活：@EnableAutoConfiguration
配置：/META-INF/spring.factories (主要是规约)
实现：XXXAutoConfiguration

嵌入式web容器
Web Servlet:Tomcat、Jetty、Undertow
Web Reactive: Netty Web Server

生产准备特性
指标：/actuator/metrics
健康检查：/actuator/health
外部化配置：/actuator/configprops

传统Servlet应用
Servlet组件：Servlet、Filter、Listener
Servlet注册：Servler注解、Spring Bean、RegistrationBean
异步非阻塞：异步Servlet（servlet 3.0）、非阻塞Servlet（servlet 3.1）

Servlet组件
1、Servlet
实现
@WebServlet 同时继承HttpServlet

URL 映射
@WebServlet(urlPatterns = "/my/servlet")

注册
@ServletComponentScan(basePackages = "com.imooc.diveinspringboot.web.servlet")

2、Filter
实现
@WebFilter 同时实现Filter

3、Listener
实现
@WebListener
同时可以选择实现一个或者多个接口
ServletContext（监听Application）
生命周期监听：ServletContextListener
属性监听：ServletContextAttributeListener

HttpSession（监听Session）
生命周期监听：HttpSessionListener
属性监听：HttpSessioniAttributeListener

ServletRequest（监听Request）
生命周期监听：ServletRequestListener
属性监听：ServletRequestAttributeListener

感知Session监听
1：HttpSessionBindingListener监听
⑴在需要监听的实体类实现HttpSessionBindingListener接口
⑵重写valueBound()方法，这方法是在当该实体类被放到Session中时，触发该方法
⑶重写valueUnbound()方法，这方法是在当该实体类从Session中被移除时，触发该方法
2：HttpSessionActivationListener监听
⑴在需要监听的实体类实现HttpSessionActivationListener接口
⑵重写sessionWillPassivate()方法，这方法是在当该实体类被序列化时，触发该方法
⑶重写sessionDidActivate()方法，这方法是在当该实体类被反序列化时，触发该方法

Spring Framework手动装配

1、模式注解装配

使用模式注解的方式，一下是常用最基础的注解
@Repository 数据仓储模式注解
@Component 通用组件模式注解
@Service 服务模式注解
@Controller Web 控制器模式注解
@Configuration 配置类模式注解

装配方式可以使用xml中<context:component-scan>方式或者@ComponentScan

  • 会扫描上述注解或者其派生的注解

    2、@Enable模块装配
    注解驱动方式：参考@EnableWebMvc

接口编程方式：参考@EnableCaching

3、条件装配
作为Bean装配的前置判断
基于配置方式@Profile 使用注解或者是配置文件的命名区分
基于编程方式@Conditional 參考：@ConditionalOnClass
